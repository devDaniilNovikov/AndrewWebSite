# Shared-memory open questions

There are no active shared-memory questions at startup.

`codex-mem` is a deferred decision, not an active question or a task blocker. It remains inactive and advisory until an explicit user-approved pilot.

## Entry template

## Q-YYYYMMDD-NNN — Short question

- **Status:** active | resolved | superseded | deferred
- **Date:** YYYY-MM-DD
- **Scope:** affected task or system boundary
- **Question:** the decision that needs an answer
- **Why now:** concrete blocked or pending work
- **Evidence:** verified facts and links, never raw transcripts
- **Canonical source:** source that will settle the question
- **Owner:** one active controller or user
- **Resolution / supersedes:** populated when no longer active
- **Review:** required reviewer or user decision

## Rules

- Add an active question only when it has a concrete owner and affects an assigned task; do not turn deferred ideas into blockers.
- Do not duplicate `TASKS.md`, live Git/GitHub state, or an executable plan.
- Resolve conflicts through status, evidence, and supersession metadata; retain the historical record.
- Move only superseded non-handoff records to `archive/` after their replacement is linked and reviewed.
