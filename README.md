# AndrewWebSite

## Спецификация проекта

AndrewWebSite — русскоязычный B2B-сайт компании по ремонту и техническому
обслуживанию торгового холодильного оборудования в Москве и ближайшем
Подмосковье.

### Цели продукта

- Основная конверсия — заметный телефонный звонок с каждой важной страницы.
- Дополнительная конверсия — короткая форма заявки.
- Принятая заявка сначала сохраняется в PostgreSQL, после чего асинхронно
  доставляется в Telegram.
- После успешной отправки формы пользователю повторно предлагается позвонить.

### Состав MVP

- главная страница `/`;
- список услуг `/uslugi/`;
- ремонт торгового холодильного оборудования;
- ремонт льдогенераторов;
- страницы компании, выполненных работ, цен и контактов;
- обязательные страницы политики конфиденциальности, согласия и других
  юридических документов.

В MVP не входят личные кабинеты, авторизация, CRM, административная панель,
интернет-магазин, платежи, бронирование, CMS, блог и микросервисная архитектура.

### Утверждённая архитектура

- Frontend: Next.js 16.2.9, React 19.2.x, строгий TypeScript, Tailwind CSS 4 и
  статический экспорт.
- Backend: Java 25 LTS, Spring Boot 4.1.0, Maven Wrapper, Spring MVC,
  Bean Validation, Spring Security, JdbcClient и Flyway.
- Хранилище: управляемый PostgreSQL 17 в том же московском регионе и VPC.
- Очередь доставки в Telegram реализуется через PostgreSQL outbox с арендой,
  повторными попытками и восстановлением после перезапуска.
- Frontend встраивается в Spring Boot JAR; production использует один Java
  container на Timeweb Cloud App Platform.
- Браузер обращается к API только через тот же origin; отдельный production
  Node.js runtime не используется.

### Приём и защита заявок

- Публичная точка приёма: `POST /api/leads` с JSON-телом.
- `requestId` обеспечивает идемпотентность; ответ `202 Accepted` возвращается
  только после успешной транзакции lead и outbox.
- Сервер проверяет входные данные, согласие, honeypot, размер запроса и лимиты
  частоты обращений.
- Персональные данные не попадают в логи, метрики, URL, аналитику или постоянное
  хранилище браузера.
- Персональные данные удаляются не позднее 30 дней после создания заявки.
- Telegram credentials и production secrets передаются только через защищённое
  хранилище секретов.

### Контентные ограничения

- Нельзя выдумывать фотографии, отзывы, цены, адреса, телефоны, реквизиты,
  гарантии, сертификаты или юридические сведения.
- До получения проверенных материалов используются явные placeholders.
- Неподтверждённые юридические тексты и production-реквизиты считаются
  блокерами запуска.
